## 7.4 Code: mycpu and myproc

### xv6 是如何让进程访问 cpuid 的？
多核情况下，会存在两个 scheduler 运行，不同 CPU 也拥有属于自己的寄存器，tp 就是其中一个
tp 在内核态和用户态时的定义不同，在用户态这用于指示正在运行的线程指针 thread pointer
然后在内核态，tp 被视为存储当前 cpu 的 hartid

tp 寄存器的更新路径为：
- 在 boot 阶段，每个 CPU 都会给自己的 tp 设置为 hartid
- 之后在每个 CPU 通过 usertrapret 调度进入用户进程时，会将tp的内容存入 trapframe->kernel_hartid
- 用户进程执行... 
- 发生 timer interrupt
- 进入内核，恢复 tp，之后被设置为 RUNNABLE
- 再次被两个 CPU scheduler 其中之一 调度
- 此时 usertrapret 中的 r_tp()，返回的就是当前 CPU的 tp了 
（在哪个 CPU 上被唤醒，r_tp()因此读取谁的tp值）
