## Page Tables

kernel 内部虚拟地址映射有两种方式，直接映射 以及 页表映射
每个进程有自己的内核栈(kstack)，并在其下方有个 guard page 用于保护栈溢出访问，guard page 本身是一段虚拟地址，没有映射到任何物理页

❗每个进程参与调度都是处于内核态，在进程列表中的进程都处于以下状态
1. 主动让出 CPU（sleep、yield），会通过系统调用进入内核态，设置状态后通过 sched 切回 schedular
2. 时间片结束，最终也会通过中断进入内核态，设置状态后也是同样 sched 切回schedular
ps：通过系统调用进入内核态的时候就会在 trampoline 中恢复 kernel pagetable，以支持用户进程在内核态下执行
因此，参与调度的进程都处于内核态，切换上下文之后还会继续以内核态执行，之后再通过 usertrapret 返回用户态，并设置用户进程的 pagetable

#### xv6 启动过程
    entry.S [设置初始内核栈] -> start() -> main() [初始化页表；创建中断响应；userinit 创建第一个进程 ...] -> scheduler()

## tools
- PS:       列出当前系统中的进程（包括僵尸进程）
- TOP:      打印当前进程资源使用的信息
- PMAP:     显示进程的内存映射
- /proc:    虚拟文件系统，以文本形式输出内核数据结构的内容 "cat /proc/loadavg"

- AR:创建静态库，插人、删除、列出和提取成员。
- STRINGS:列出一个目标文件中所有可打印的字符串。
- STRIP:从目标文件中删除符号表信息
- NM:列出一个目标文件的符号表中定义的符号。
- SIZE:列出目标文件中节的名字和大小。
- READELF:显示一个目标文件的完整结构，包括ELF头中编码的所有信息,
- OBJDUMP:所有二进制工具之母。能够显示一个目标文件中所有的信息它量的作用是反汇编.text节中的二进制指令。
- LDD，列出一个可执行文件在运行时所需要的共享库。
