## pipe

```
pipe(p);
if(fork() == 0) {
    close(0);
    dup(p[0]); // 重定向，使得标准输入指向管道一侧
    close(p[0]); // 构建上面关系后，管道的指向就可以关闭了，保证了管道的一个读端
    close(p[1]);
    exec("/bin/wc", argv); // 执行 wc 之后，会 read from pipe，如果此时没有内容，则阻塞等待
} else {
    close(p[0]); // 写端不需要读
    write(p[1], "hello world\n", 12); // 这里直接往管道指向内容进行写入，而不需要重定向
    close(p[1]); 
}

```

- 管道实际就是父子进程都引用同一个文件表项（读写各一个表项），但是各自都有一份描述表
- 子进程在重定向之后要 close 自己的管道的写描述符，保证管道最终不会有写端，否则父进程 write 结束后，子进程后续会永久阻塞，无法释放
