## 1️⃣ `ps` 中 `STAT` 字段含义

`ps aux` 输出中，`STAT` 表示进程状态，通常是 **一个或多个字符**，常见的有：

| 符号    | 含义                                     |
| ----- | -------------------------------------- |
| **R** | Running：正在运行或在运行队列中等待 CPU              |
| **S** | Sleeping：休眠中，等待某个事件（可中断）               |
| **D** | Uninterruptible sleep：不可中断休眠（通常等待 I/O） |
| **T** | Stopped：进程被停止（如 `Ctrl+Z` 或信号停止）        |
| **Z** | Zombie：僵尸进程，子进程已结束，但父进程尚未回收            |
| **X** | Dead：死亡进程，内核标记为死掉（很少见）                 |
| **I** | Idle：空闲内核线程                            |

此外，状态可能有附加字符：

| 符号    | 含义                     |
| ----- | ---------------------- |
| **+** | 位于前台进程组（terminal）      |
| **<** | 高优先级进程                 |
| **N** | 低优先级进程                 |
| **L** | 有些页面被锁入内存              |
| **s** | 是会话首进程（session leader） |
| **l** | 多线程（使用 CLONE_THREAD）   |

> 小提示：你看到 `S+` 就表示 “可中断休眠且在前台”。
